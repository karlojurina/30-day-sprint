# EcomTalent — Design Context

This file is the source of truth for every design decision in the EcomTalent
student platform. Read it before any design work so nothing drifts toward
generic AI / generic SaaS output.

## Design Context

### Users

Paid EcomTalent members. They bought a $97/month seat on Whop to learn how to
create ads that get brands paid, and to land real paid work by the end of the
30-day sprint. Age range skews 18–30, mixed technical skill, global. They
open this platform during focused work blocks — usually at a desk, often in the
evening. Mobile is secondary; **desktop-first** is the default assumption.

The single job this interface exists to do: **tell the student exactly what to
do next, make them feel like they're investing in themselves, and respect their
time on the way.**

### Brand Personality

Three words from Karlo (the founder): **locked in, valuable, chill.**

- **Locked in** — disciplined, intentional, focused. No bouncing, no
  distractions, no "what if we tried X?". The student is here to do work.
- **Valuable** — premium. Worth the $97. Respects what they paid for. Not
  budget, not free-tier aesthetic.
- **Chill** — confident enough not to shout. Never frantic. Scarcity and
  urgency are stated plainly, never screamed. The interface has nothing to
  prove.

Voice on copy: direct, mentor-like, no fluff. Karlo is the voice throughout.

### Aesthetic Direction

Committed direction: **an explorer's sea chart. Parchment, ink, compass, fog
of war.** The 30-day program is a map you're charting as you go. Regions are
terrains you cross. Progress is ink on vellum.

This is **not gamer, not arcade, not neon**. Think: a museum caption, the
inside cover of a 1950s navigation handbook, a hand-drawn chart on aged paper
that happens to be lit from behind.

**Committed palette (OKLCH where possible, hex fallbacks documented):**

| Role | Hex | Use |
|---|---|---|
| Deep sea navy (primary bg) | `#060C1A` | App background, darkest surface |
| Navy secondary | `#0A1428` | Panels behind the map |
| Navy card | `#102042` | Cards, modals, topbar chips |
| Navy elevated | `#15294C` | Hover / elevated cards |
| Parchment gold (primary accent) | `#E6C07A` | Path, completed nodes, CTAs, "you are here" |
| Gold light | `#F0D595` | Hover / active gold |
| Gold deep | `#C89A4A` | Pressed / outer medallion ring |
| Teal (brand + milestones) | `#4DCEC4` | Peer presence, watch-type indicators |
| Sky (secondary info) | `#4DA0D8` | Action-type indicators |
| Warm crimson | `#C44A54` | **Boss node ONLY.** Never decorative. |
| Bone ink (text) | `#E6DCC8` | Primary text — warm off-white, never pure white |

Neutrals are tinted toward gold (`rgba(230,220,200,0.xx)` scales for borders
and secondary text). Pure black and pure white never appear.

**Committed typography:**

| Role | Font | Usage |
|---|---|---|
| Display | Cormorant Garamond, 500, **italic** | Region names, lesson titles, modal headings, ceremonial copy |
| Body | Inter | All body copy, form inputs, buttons |
| Mono | JetBrains Mono | Meta labels ("DAY 05 · REGION II"), day numbers, short uppercase tags |

Mono utility: 11px, 500, `letter-spacing: 0.16em`, uppercase, bone-ink-faint
color. Used for small meta labels only — never body text, never whole
paragraphs.

Display line-height: 1.02 (tight). Italic serif for display creates editorial
cadence that separates this from every dashboard-style SaaS on the internet.

**Committed textures & effects:**

- Paper grain overlay: full-viewport SVG noise, `mix-blend-mode: overlay`,
  opacity 0.7, above everything at z-500. Do not remove.
- Vignette frame around the map.
- Compass rose, scale bar, corner ornaments, cartouches on the map.
- Motion easing is always `cubic-bezier(0.22, 1, 0.36, 1)` (ease-out-quart).
  Durations: 0.2s for quick feedback, 0.4s for modal fade, 0.5s for sheet
  slides, 0.6s for celebrations. **Never** bounce or elastic.

### Light Mode

**There is no light mode.** The entire product ships dark. Admin, login,
dashboard, public landing if it ever exists — all dark-on-parchment-gold. The
explorer's chart doesn't work under fluorescents; don't build a light theme.

### Accessibility

No special requirements stated. Apply the baseline:

- Body text contrast ≥ 4.5:1 against background.
- Interactive states have a focus ring (2px gold at 0.5 opacity).
- `prefers-reduced-motion` disables the pulsing rings and auto-animations
  (kept as a simple color change / static state).
- Target sizes ≥ 40px on touch.

### Design Principles

1. **The map is the product.** Every screen reinforces the expedition
   metaphor: regions, charts, "charted", "region locked". Don't introduce
   generic SaaS patterns that break the illusion (toast stacks, right-side
   drawer modals, Intercom-style chat bubbles, blue progress bars).

2. **Calm confidence over urgency.** Countdowns are stated factually, not
   shouted. "14 days left" lives in the topbar as mono text; it never flashes,
   never pulses red. Scarcity lands because the student respects the source,
   not because we yelled.

3. **Ink-on-parchment, not glass-on-neon.** No glassmorphism. No glowing
   accents for their own sake. No gradient text (hard ban). No purple/cyan AI
   palette. Pulsing glow is permitted exclusively on (a) the "you are here"
   marker, (b) the discount gate node, (c) the current-day pin — because those
   three points actively guide the eye.

4. **Typography does the hierarchy, not color.** Display italic vs mono tiny-
   caps vs Inter body is enough hierarchy for most cases. Color is used for
   state (done/current/locked/boss), not for visual interest.

5. **Respect the student's time.** One clear next action at all times. No
   empty states that say "nothing here yet." No redundant copy (meta line
   above title should never restate the title). No button that just says
   "Dismiss." Every word earns its place.

6. **Dark everywhere.** No light-mode fallbacks. No "toggle theme" chip in
   settings.

### Non-negotiable Bans (extend the global impeccable bans)

In addition to the universal impeccable bans (side-stripe borders, gradient
text, AI palette, etc.), this project forbids:

- **Bouncy / elastic easing.** Always use ease-out-quart.
- **Right-side drawer modals for primary content.** Use centered modals.
  The right drawer felt thin and we reverted it.
- **Generic checkpoint iconography** (trophy cups, chat bubbles, gear icons).
  If a node needs an icon, use the shapes defined in `map/LessonNode.tsx`
  (circle = watch, diamond/rotated-square = action, 16-point star = gate,
  8-point star = boss). Consistency carries the metaphor.
- **Sans-serif for ceremonial copy.** Use Cormorant Garamond italic.
- **Sync buttons, refresh buttons, any UI that asks the student to manually
  pull data.** The platform syncs silently in the background.
- **"Let's get started" / "You're awesome!" / "Welcome back, champion!"**
  copy. No LinkedIn energy. Karlo is direct; the copy is direct.
